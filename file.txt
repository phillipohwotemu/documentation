version: '3.7'

      - ~/jenkins_home:/var/jenkins_home

SonarQube has some hardware requirements : you should increase the values for those 2 OS settings :

fs.file-max: â€œThe value in file-max denotes the maximum number of file- handles that the Linux kernel will allocate. When you get lots of error messages about running out of file handles, you might want to increase this limit.â€
vm.max_map_count: â€œthe maximum number of memory map areas a process may have. Memory map areas are used as a side-effect of calling malloc, directly by mmap and mprotect, and also when loading shared libraries.â€
You should modifiy as root user the /etc/sysctl.conf file to set permanently the values for these 2 parameters :

$> sudo nano /etc/sysctl.conf
Then add this content to the file :

# can be added at the end of the file
fs.file-max=131072
vm.max_map_count=524288
Save, close, and then reload the settings :

$> sudo sysctl --system
